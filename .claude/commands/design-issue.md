당신은 현재 리포지토리의 이슈 트래킹을 돕는다. 사용자가 준 **이슈 설명**을 받아, 잘 구성된 GitHub 이슈를 계획하고 생성한다.

이슈 설명: $ARGUMENTS

## 목표

애매한 아이디어를 명확한 GitHub 이슈로 만든다. 무엇을 해결하려는지 불명확하면 **먼저 사용자에게 물어보고**, 명확해지면 표준 템플릿으로 이슈를 만든다.

## 실행 순서

### 1. 리포 확인
- 현재 디렉토리가 대상 프로젝트의 git 리포인지 확인한다. `gh`는 이 디렉토리 안에서 실행하면 git 리모트로부터 **호스트와 owner/repo를 자동 판별**하므로, `GH_HOST`·`--repo`를 붙이지 않는다. (엔터프라이즈 호스트도 해당 호스트에 `gh auth login`이 되어 있으면 자동 인식된다.)
- 예외: 프로젝트 디렉토리 밖에서 실행하거나 리모트로 판별이 안 되면, 그때만 `--repo <owner/repo>`(엔터프라이즈면 `GH_HOST=<호스트>`도)를 명시한다.

### 2. 문제 정의 (이 이슈의 핵심)
이슈의 목적은 **문제를 규정하는 것**이지 구현 방법을 적는 게 아니다. 설계가 명확해지면 구현은 그 뒤에 자연히 따라오므로, 구현 작업 목록은 이슈의 스코프가 아니다. 아래 네 가지를 규정할 수 있는지 검토한다.
- **문제점**: 지금 무엇이 문제인가 / 왜 지금 다뤄야 하는가.
- **해결해야 할 포인트**: 이 이슈에서 반드시 풀어야 할 핵심 논점·질문.
- **트레이드오프**: 선택지 사이에서 저울질해야 할 요소, 제약 조건.
- **지표**: 무엇으로 해결·개선을 판단하는가 (가능하면 측정 가능한 기준).

이 중 하나라도 설명만으로 규정하기 어려우면 **AskUserQuestion으로 되묻는다.** 네 가지가 충분히 잡힐 때까지 반복하고, 사용자가 추측해도 된다고 말하지 않는 한 애매한 채로 이슈를 만들지 않는다.

### 3. 리포 컨텍스트 탐색
- 리포의 문서와 관련 소스를 훑어, 이슈 배경과 **참고 링크**(관련 문서·코드 경로)를 보강한다.
- 있으면 `README`, `docs/`, `AGENTS.md`/`CLAUDE.md`/`CONTRIBUTING.md` 같은 규약 문서를 우선 확인하고, 그 리포가 정한 문서/아키텍처 권위 규칙을 따른다. 없으면 소스 구조에서 직접 파악한다.

### 4. 유사 이슈 중복 검색
- `gh issue list --state all --search "<핵심 키워드>"`로 유사 이슈를 찾는다.
- 강하게 유사한 이슈가 있으면 **목록을 사용자에게 보여주고**, 다음 중 무엇을 할지 확인한다:
  - 새 이슈를 그대로 만든다
  - 기존 이슈에 코멘트를 단다 (그렇다면 코멘트 내용을 제안하고 확인 후 게시)
  - 중단한다
- 유사 이슈가 없으면 그대로 진행한다.

### 5. 이슈 초안 작성
아래 **표준 템플릿**으로 제목과 본문을 작성한다. 본문 언어는 이슈 설명의 언어에 맞춘다(기본 한국어). 각 섹션은 2단계에서 규정한 문제 + 3단계에서 모은 컨텍스트로 채운다. 해당 없는 항목은 억지로 채우지 말고 섹션 자체를 최소화한다. **구현 방법·작업 목록은 넣지 않는다** — 이 이슈는 문제와 판단 기준을 규정하는 데 집중한다.

```markdown
## 배경 / 문제
<지금 무엇이 문제인가, 왜 지금 다뤄야 하는가>

## 해결해야 할 포인트
- <이 이슈에서 반드시 풀어야 할 핵심 논점·질문>
- ...

## 고려할 트레이드오프
- <선택지 간 저울질할 요소, 제약 조건 — 예: A vs B의 득실>

## 지표 / 판단 기준
- <무엇으로 해결·개선을 판단하는가 (가능하면 측정 가능한 기준)>

## 참고
- <관련 문서 경로, 링크, 유사 이슈 번호>
```

제목은 문제가 드러나는 간결한 한 줄로 짓는다.

### 6. 초안 확인 후 생성
- 완성한 제목·본문 초안을 사용자에게 보여준다.
- **사용자 승인을 받은 뒤에만** 이슈를 생성한다 (이슈 생성은 외부 게시 행위다. 승인 없이 생성하지 않는다).
- 승인되면:
  ```
  gh issue create --title "<제목>" --body "<본문>"
  ```
- 생성된 이슈 URL을 사용자에게 보고한다.

## 하지 않는 것
- 코드 구현/변경, 브랜치 생성, commit·push는 하지 않는다. 이 명령은 **이슈 생성까지만** 한다.
- 라벨/담당자는 사용자가 요청할 때만 지정한다.
